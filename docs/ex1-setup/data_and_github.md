## Data and GitHub
You can get all the exercise material including the data by (for example):

- **Option 1**: download the current version of the data as a `.zip` file. Do this by going to the top of any exercise page on this website and click _Download Data_ (For example [here](../ex1-IntroductionToImageAnalysis/README/#getting-the-data)).

- **Option 2**: download the _current_ version of the exercise and data as a `.zip` file. Do this by clicking here: [All Data](https://github.com/RasmusRPaulsen/DTUImageAnalysis/archive/refs/heads/main.zip). You can also go to the [main page](https://github.com/RasmusRPaulsen/DTUImageAnalysis/tree/main) of the 02503 Image Analysis GitHub repository and pressing the green `<>Code` button and select the **Download ZIP** option.

- **Option 3**: Clone the repository using a [Git](https://git-scm.com/) client. Assuming you've setup Git beforehand, go to a terminal and type:
    ```Shell
    cd /path/to/exercise/folder

    git clone https://github.com/RasmusRPaulsen/DTUImageAnalysis.git
    ```

Git allows you to keep up to date with the course material in an easy fashion. If you currently haven't setup Git and would like to do so, we recommend you take a look at the [Official "Set up Git" tutorial](https://docs.github.com/en/get-started/git-basics/set-up-git).


### Rolling updates
Throughout the course, updates to the course material will occur. This will mostly happen periodically, as exercise solutions are released some time after a given exercise session. Irregular updates may also occur, which you should look out for.

If you're using Git to manage course files, you may want to pull new updates somewhat regularly, e.g. weekly. This can be done by running:

```shell

cd /path/to/exercise/folder

git pull

```

If you're not using Git, we advise you to download the material from the webpage during each exercise session ([Option 1](#data-and-github)). At the end of the course, you would then download the whole updated material again from the repository as a `.zip` file ([Option 2](#data-and-github)).
